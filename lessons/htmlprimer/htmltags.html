1. HTML Headings

As we briefly touched in the introduction section that we need to cater for Headings and Paragraphs to structure text properly. HTML has multiple tags to cater for this. For Headings there are six level of heading tags which HTML provides, and they are as follows:

  • <H1> This is Heading Level 1 </H1>
  • <H2> This is Heading Level 2 </H2>
  • <H3> This is Heading Level 3 </H3>
  • <H4> This is Heading Level 4 </H4>
  • <H5> This is Heading Level 5 </H5>
  • <H6> This is Heading Level 6 </H6>

Structure Hierarchy: It is important to maintain the structure hierarchy when we are laying down the text content on our webpages. If we think about following are the different areas of users and applications that will access our webpage, and we as developers need to make sure that content is structured in such a way that for all the below use cases it can be easily and intuitively accessed:

  • Search Engine: the crawlers of search engines when index our webpages, need to understand the content on it and in doing so the headings plays important part in it amongst various other factors involved in Search Engine Optimization. Without headings our webpage may not perform well in SEO and be found organically on the web.
  • CSS Javascript: we also need to structure our webpages properly so that we can use CSS to select the contents based on the class and variour other selection and apply appropriate styling. Proper structure of HTML pages is also required so that we can access the content programatically and it requires proper tags, ids and classes etc to be in place for the elements to be properly targeted.
  • Users: We, the end users, usually do a quick scan of any webpage before we decide to spend time on a page for a detailed read. So it is important to have a proper structure of headings, contents, lists, quotes, etc in any document and HTML pages are no exception to it.
  • Accessibility: We should also not forget that wide variety of users who access websites may have some visual impairement, so our websites should also cater to it as they will use Screen Readers to read the content of website. In that regards it is important that we use proper tags for e.g. <emphasis> and <strong> wherever require. Also a proper use of semantic tags is advisable. We have a section on Semantic Tags later in our course.
